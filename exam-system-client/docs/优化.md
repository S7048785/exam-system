## 前端 TanStack Start 项目缺陷分析报告

经过对代码库的全面审查，共发现 **50+ 个问题**，按严重程度分类如下：

---

### 🔴 严重缺陷（Critical）— 需立即修复

#### 1. QueryClient 每次导航重建，缓存完全失效

**文件:** `src/integrations/tanstack-query/root-provider.tsx:3-16`, `src/router.tsx:6-15`

`getContext()` 每次调用都 `new QueryClient()`，导致每次路由切换产生新实例，所有 staleTime/gcTime 缓存配置形同虚设。这是根本性架构问题。

#### 2. 多个路由 loader 缺少 `await`，数据预取形同虚设

- `src/routes/admin/questions.tsx:7-18` — `Promise.all([...])` 未 await
- `src/routes/admin/banners.tsx:7-10` — `ensureQueryData` 未 await
- `src/routes/admin/categories.tsx:7-9` — 同上
- `src/routes/admin/papers/list.tsx:7-9` — 同上

loader 不等待数据就绪，`useSuspenseQuery` 会触发 Suspense 悬挂，pending 状态也不生效。

#### 3. 登录/注册表单 `isError` 检查在 `mutateAsync` 之后是死代码

**文件:** `src/features/login/LoginForm.tsx:56-67`, `src/features/login/RegisterForm.tsx:37-47`

`mutateAsync` 在错误时会 throw，永远不会执行到后面的 `if (loginMutation.isError)` 分支。表单重置逻辑在失败时不会执行。

#### 4. 选择题分析数据丢失 Bug

**文件:** `src/features/admin/questions/QuestionDrawer.tsx:134`

`ChoiceForm` 的分析编辑通过 `choiceData.analysis` 更新，但提交时用的是 `value.analysis`（表单级字段），UI 编辑的分析永远不会被提交。

#### 5. Exam 搜索 queryKey 不含参数，搜索完全失效

**文件:** `src/features/exam/list/examQueries.ts:12-13`

`queryKey: ['paperList']` 不包含 `param.name` 和 `param.status`，切换搜索关键词不会触发新的请求。

#### 6. API 非 200 响应时 JSON 解析可能崩溃

**文件:** `src/ApiInstance.ts:47-50`

`throw await response.json()` 在后端返回非 JSON 内容（如 502 HTML）时抛出 `SyntaxError`，原始错误信息丢失。

---

### 🟠 高危问题（High）

#### 7. 试卷分页参数未传递给 API

**文件:** `src/features/admin/papers/index.tsx:17-23`

组件维护了 `page`/`size` state，但从未传给 `papersQueryOptions`。`total` 计算只用当前页数组长度，不是真正的总数。

#### 8. Admin 路由守卫仅客户端执行，`ssr: false`

**文件:** `src/routes/admin/route.tsx:4-16`

`ssr: false` 完全禁用 SSR，路由守卫只在客户端执行，可被绕过。真正的权限校验必须在后端 API 层。

#### 9. 多处 `data.data!` 非空断言，API 返回 null 时崩溃

- `src/features/admin/banners/index.tsx:51`
- `src/features/exam/list/index.tsx:42`
- `src/features/admin/papers/components/RuleCreatePage.tsx:46-48`（硬编码索引 `[0]`/`[1]`/`[2]` 更危险）

#### 10. `defaultPreloadStaleTime: 0` 使预加载无效

**文件:** `src/router.tsx:14`

鼠标悬停触发预加载，但缓存立即过期，大量不必要的 API 请求。

#### 11. 选择题正确答案索引偏移 Bug

**文件:** `src/features/admin/questions/QuestionDrawer.tsx:124-127`

用 `filledChoices`（过滤后）的索引映射字母，如果用户留空某个选项，字母映射会错位。

#### 12. Papers queryKey 每次渲染创建新对象引用

**文件:** `src/features/admin/papers/index.tsx:22`

`papersQueryOptions({ ...filters })` 每次渲染创建新对象，queryKey 变化导致每次都重新请求。

### 🟡 中等问题（Medium）

#### 13. 多处 toast 在 mutation 完成前显示

- `BannerTable.tsx:43-51, 57-64` — 新增/更新/删除操作
- `CategoryTable.tsx:192-195` — 删除操作
- `QuestionTable.tsx:320-323` — 删除操作

#### 14. logout 未 await，可能导致登出不完整

**文件:** `src/components/layout/Header.tsx:18-24`

`api.userController.logout()` 缺少 await，后续的 invalidate 可能在后端清除 cookie 前执行。

#### 15. 多处 useEffect 缺少依赖

- `QuestionDrawer.tsx:207` — 缺少 `form`
- `ChoiceForm.tsx:106`, `JudgeForm.tsx:29`, `TextForm.tsx:18` — 缺少 `onChange`

#### 16. 侧边栏用户信息硬编码为假数据

**文件:** `src/features/admin/app-sidebar.tsx:23-28`

始终显示 `shadcn` / `m@example.com`，未接入真实用户数据。

#### 17. `RuleCreatePage` 硬编码分类索引

**文件:** `src/features/admin/papers/components/RuleCreatePage.tsx:44-51`

假设 `data[0]` 是选择题、`data[1]` 是判断题、`data[2]` 是简答题，后端顺序变化即崩溃。

#### 18. 编辑页面"取消"导航到创建页而非列表页

**文件:** `src/features/admin/papers/components/CreateEditPage.tsx:175-179`

#### 19. 表单验证不完整

- Banner 表单零验证
- 注册表单 `realName` 限制恰好 2 字符（中文名 2-4 字符很常见）
- `email` 字段未做邮箱格式校验
- 分数字段接受 NaN 和 0

#### 20. `<a href>` 替代 `<Link>` 导致全页面刷新

- `src/components/layout/Header.tsx:70-74`
- `src/routes/_public/index.tsx:20-25`

### 🔵 低危 / 代码质量问题

#### 21. 页面标题仍是模板默认值 "TanStack Start Starter"

**文件:** `src/routes/__root.tsx:33`

#### 22. 多处 `console.log` 调试代码未清理

- `BannerTable.tsx:60`, `TableCellViewer.tsx:50`, `QuestionDrawer.tsx:142`, `RuleCreatePage.tsx:83`

#### 23. 删除确认弹窗的"取消"按钮无功能

- `BannerTable.tsx:134-136`, `CategoryTable.tsx:186-188`, `QuestionTable.tsx:314-315`

#### 24. `admin/about` 是模板残留（Minecraft 3D 模型查看器）

#### 25. 英文硬编码混入中文 UI

- `TableCellViewer.tsx:70` — "Showing total visitors..."
- `exam/list/index.tsx:63,65` — "Search..." / "Search"

#### 26. 多个 TanStack 包使用 `"latest"` 版本标签

**文件:** `package.json:27-34`

构建不可复现，任何 breaking change 会静默破坏项目。

#### 27. `tailwind.config.js` 是死代码

项目用 Tailwind v4（CSS-first 配置），此文件是 v3 格式，且 `content` 漏了 `.ts`/`.tsx`。

#### 28. 无代码分割配置，大型依赖全打包

`recharts`（~400KB）、`motion`（~100KB）、`react-skinview3d` 等均无 lazy load。
